# PRD - 成电吃什么 Agent (MVP)

## 1. 产品背景
电子科技大学学生在校内或校周边就餐时，经常面临信息分散、选择困难、时间紧张的问题。
本项目通过自然语言输入快速给出 3 家可执行的餐饮推荐。

## 2. MVP 目标
- 用户输入一段自然语言需求。
- 系统抽取核心条件：预算、校区/位置、场景、口味、时间。
- 基于本地 mock 店铺数据筛选和排序。
- 返回 3 家推荐店铺及推荐理由。
- 预留讯飞模型接入层，当前默认使用规则解析。

## 3. 目标用户
- 电子科技大学清水河/沙河校区学生。
- 典型场景：课间快餐、社团聚餐、夜宵、约饭。

## 4. 用户输入与输出
### 输入（自然语言）
示例：
- “预算 30，清水河这边，晚上想吃辣的，和室友一起。”
- “沙河校区中午，一个人，清淡点，20 以内。”

### 输出（推荐结果）
- 结构化解析结果（budget/location/scene/taste/time）
- 3 家推荐店铺，每家包含：
  - 店名
  - 人均价格
  - 距离或位置
  - 标签
  - 推荐理由（与用户条件匹配点）

## 5. 功能范围
### In Scope
- 单轮问答推荐
- 本地 CSV 数据
- 简单规则抽取 + 打分排序
- REST API + Web 页面

### Out of Scope（MVP 暂不做）
- 用户登录和历史画像
- 实时外卖/排队数据
- 多轮对话记忆
- 地图可视化

## 6. 非功能需求
- 响应时间：本地数据下 < 1s
- 可扩展：后续可替换为 SQLite/向量检索/大模型
- 代码可读性：模块化、低耦合

## 7. 成功指标（MVP）
- 能解析 80% 常见输入中的核心槽位
- 每次稳定返回 3 家推荐（数据不足时返回可用数量并说明）
- 推荐理由可读且与输入条件相关
